# Map Consistency Checker

## 概述

`map-consistency-checker` 负责检查章节中角色移动、行动、行走等描述是否与地图设定集保持一致。

## 检查范围

### 1. 距离一致性
- 角色移动距离是否与地图标注的相对位置匹配
- 移动时间是否与距离和移动方式匹配
- 禁止出现"片刻/呼吸"跨越长距离

### 2. 方向一致性
- 移动方向是否与地图方向一致（如城东、城西）
- 位置关系描述是否正确
- 禁止方向错误（如城东写成城西）

### 3. 时间一致性
- 移动时间是否符合移动方式的速度
- 凡人/修士移动速度是否合理
- 境界与移动方式是否匹配

### 4. 路径合理性
- 两点间移动是否有合理路径
- 是否跳过必经的障碍物
- 道路网络是否被正确使用

## 检查规则

### 距离检查规则

| 场景 | 阈值 | 说明 |
|------|------|------|
| 城内短距离 | 1-2里 | 可用"片刻/呼吸" |
| 城内区域间 | 3-10里 | 需"一盏茶/小半个时辰" |
| 城郊 | 10-30里 | 需"大半个时辰/一个时辰" |
| 城际 | 30-100里 | 需"半日/一日" |
| 跨区域 | 100里以上 | 需"数日" |

### 方向检查规则

- 城东、城西、城南、城北方向必须与地图一致
- 位置描述（如"A在B的东边"）必须正确
- 区域划分（如"东城区"、"西城区"）必须存在

### 时间检查规则

凡人速度：
- 步行：5-10里/时辰
- 快马：50-80里/时辰

修士速度（修仙）：
- 炼气：50里/时辰
- 筑基：200里/时辰
- 金丹：500里/时辰
- 元婴：1000里/时辰

### 路径检查规则

- 禁止直接穿越山脉、湖泊等自然障碍
- 必须使用道路网络（除非有特殊能力）
- 传送阵/瞬移必须有设定支持

## 审查输出格式

### 发现问题时

```json
{
  "checker": "map-consistency-checker",
  "severity": "critical|high|medium|low",
  "issues": [
    {
      "type": "distance|direction|time|path",
      "location": "第X段/第Y行",
      "description": "问题描述",
      "expected": "正确描述",
      "actual": "实际描述",
      "fix_suggestion": "修复建议"
    }
  ],
  "overall_score_impact": -5
}
```

### 无问题时

```json
{
  "checker": "map-consistency-checker",
  "status": "pass",
  "issues": [],
  "overall_score_impact": 0
}
```

## 报告示例

### 审查报告中体现

```markdown
## 地图一致性检查

### 🔴 距离问题（高优先级）

| 位置 | 问题描述 | 修复建议 |
|------|---------|---------|
| 第3段 | "只是几个呼吸便从城东到了城西" | 改为"耗费了小半个时辰"或"御剑飞行" |

### 🟠 方向问题（中优先级）

| 位置 | 问题描述 | 修复建议 |
|------|---------|---------|
| 第5段 | "往北望去是大海" | 北方应是山脉，检查地图设定 |

### 🟡 时间问题（低优先级）

| 位置 | 问题描述 | 修复建议 |
|------|---------|---------|
| 第8段 | "日行千里的神驹走了三天" | 改为"一日便到" |
```

## 修复优先级

1. **Critical（必须修复）**：
   - 方向完全错误（如东写成西）
   - 距离严重不符（千里距离写成片刻）

2. **High（建议修复）**：
   - 距离略有出入
   - 时间与距离不匹配

3. **Medium（可选优化）**：
   - 移动方式描述不够精确
   - 路径描述不够详细

4. **Low（忽略）**：
   - 细微的修辞问题
   - 不影响理解的描述
