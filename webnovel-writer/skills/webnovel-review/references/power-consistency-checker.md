# Power Consistency Checker

## 概述

`power-consistency-checker` 负责检查章节中功法、法术、阵法、秘术、技巧等能力的使用是否与设定集保持一致。

## 检查范围

### 1. 能力学习门槛
- 角色使用的功法/法术是否与其境界匹配
- 是否有越阶使用能力的情况
- 越阶使用是否有合理理由

### 2. 能力威力一致性
- 能力展现的威力是否与设定匹配
- 越阶击败敌人是否有伏笔
- 能力克制关系是否正确

### 3. 消耗合理性
- 高阶法术的法力消耗是否合理
- 禁术/秘术的代价是否正确描写
- 冷却时间是否存在

### 4. 克制关系
- 属性克制是否正确（水克火、火克木等）
- 境界压制是否体现
- 秘术/阵法的特定克制是否正确

### 5. 能力获得途径
- 新能力是否有获得描写
- 传承/领悟/学习是否有伏笔
- 禁止凭空出现新能力

## 检查规则

### 境界与能力匹配

| 角色境界 | 可使用能力上限 |
|---------|---------------|
| 练气期 | 基础法术、基础技巧 |
| 筑基期 | 中级法术、中级技巧、基础阵法 |
| 金丹期 | 高级法术、高级技巧、中级阵法 |
| 元婴期 | 顶级秘术、高级阵法 |
| 化神期 | 禁术、顶级阵法 |

### 越阶战斗检查

| 场景 | 允许条件 |
|------|---------|
| 越一阶击败 | 金手指/秘术爆发/对方有伤/属性克制 |
| 越两阶击败 | 主角特有优势/剧情需要/对方轻敌 |
| 越三阶击败 | 极少，需特殊理由 |

### 消耗检查

- 高阶法术需消耗对应境界的法力
- 禁术需有代价（气血/寿元/根基）
- 连续使用强力技能需有冷却

### 属性克制表

| 攻击属性 | 被克制 | 克制 |
|---------|-------|------|
| 火 | 水 | 木 |
| 水 | 土 | 火 |
| 木 | 金 | 土 |
| 金 | 火 | 木 |
| 土 | 木 | 金 |
| 冰 | 火/雷 | 水 |
| 雷 | 土 | 无特定 |
| 光 | 暗 | 暗 |
| 暗 | 光 | 光 |

## 审查输出格式

### 发现问题时

```json
{
  "checker": "power-consistency-checker",
  "severity": "critical|high|medium|low",
  "issues": [
    {
      "type": "threshold|power|consumption|restraint|acquisition",
      "location": "第X段/第Y行",
      "description": "问题描述",
      "expected": "正确描述",
      "actual": "实际描述",
      "fix_suggestion": "修复建议"
    }
  ],
  "overall_score_impact": -5
}
```

### 无问题时

```json
{
  "checker": "power-consistency-checker",
  "status": "pass",
  "issues": [],
  "overall_score_impact": 0
}
```

## 报告示例

### 审查报告中体现

```markdown
## 功法能力一致性检查

### 🔴 境界不匹配（高优先级）

| 位置 | 问题描述 | 修复建议 |
|------|---------|---------|
| 第3段 | 练气期主角使用金丹期法术"天雷崩" | 改为筑基期可用的中级雷法 |
| 第5段 | 筑基期越两阶击败金丹期修士 | 添加主角有秘境传承优势 |

### 🟠 威力不符（中优先级）

| 位置 | 问题描述 | 修复建议 |
|------|---------|---------|
| 第8段 | "天雷掌只是普通雷法" | 改为"天雷掌乃雷系顶级法术" |

### 🟡 消耗缺失（低优先级）

| 位置 | 问题描述 | 修复建议 |
|------|---------|---------|
| 第10段 | 使用禁术无代价描写 | 添加"面色苍白"等代价描写 |
```

## 修复优先级

1. **Critical（必须修复）**：
   - 低境界使用高阶能力无理由
   - 越三阶击败敌人无任何伏笔
   - 能力凭空出现无获得途径

2. **High（建议修复）**：
   - 越一阶/两阶击败理由不充分
   - 威力描述与设定不符

3. **Medium（可选优化）**：
   - 消耗描写不详细
   - 克制关系未体现

4. **Low（忽略）**：
   - 细微的修辞问题
   - 不影响理解的描述
