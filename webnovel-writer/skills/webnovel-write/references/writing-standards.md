# 写作规范

本文档定义了小说写作中的基本规范，用于确保文章质量并优化 token 使用。

## 1. 换段规范

### 1.1 换段原则

**核心原则**：严格控制文章换段，每一段内容应该包含完整的叙事单元。

**换段条件**（满足其一即可换段）：
1. **场景切换**：地点发生变化（如：从室内到室外、从城内向城外）
2. **时间跳转**：时间发生明显变化（如：从白天到夜晚、从今日到明日）
3. **视角切换**：叙述视角发生变化（如：从主角视角切换到配角视角）
4. **情节递进**：故事情节发生明显推进或转折
5. **信息分层**：不同类别的信息需要分开叙述（如：对话与描写分开）

### 1.2 禁止换段的情况

**以下情况不应换段**：
- 简单的一句话描述就换段
- 连续的对话没有行动描写就换段
- 同一场景内的普通描写频繁换段
- 只有一句话的过渡段落

### 1.3 换段示例

**错误示例**：
```markdown
他走进了房间。
房间很简陋。
只有一张床。
和一张桌子。
```

**正确示例**：
```markdown
他推开门走进了房间。房间很简陋，除了一张床和一张桌子，再无其他家具。墙面斑驳，像是多年未曾修缮，显然这里已经很久没有人居住。
```

**错误示例**（对话换段）：
```markdown
"你来了。"他说。
"是的。"我回答。
```

**正确示例**：
```markdown
"你来了。"他抬起头，目光平静。
"是的。"我点头应道，心中却有些忐忑。
```

### 1.4 段落长度建议

| 场景类型 | 建议字数 | 说明 |
|---------|---------|------|
| 动作场景 | 100-300字 | 快节奏，可稍短 |
| 对话场景 | 150-400字 | 对话+动作描写 |
| 心理描写 | 80-200字 | 内心活动描写 |
| 场景描写 | 100-300字 | 环境/氛围描写 |
| 过渡场景 | 50-150字 | 简洁过渡 |

### 1.5 换段检查清单

写作时检查以下问题：
- [ ] 这个段落是否包含完整的叙事信息？
- [ ] 换段是否因为场景/时间/视角变化？
- [ ] 是否有"一句话段落"需要合并？
- [ ] 段落长度是否在合理范围内？
- [ ] 连续的短段落是否可以合并？

## 2. 引号使用规范

### 2.1 必须使用引号的情况

以下内容**必须**使用引号包裹：

1. **对话**
   ```markdown
   "今天天气真好。"张三说道。
   ```

2. **内心独白**
   ```markdown
   "没想到他竟然会在这里出现。"她暗自想到。
   ```

3. **演讲/宣言**
   ```markdown
   "今日之战，有我无敌！"他高举长剑，声音洪亮。
   ```

4. **引用/书名/口号**
   ```markdown
   门上写着"浩然正气"四个大字。
   ```

5. **心理活动（直接内心独白）**
   ```markdown
   "我一定要拿到第一名！"他暗暗攥紧了拳头。
   ```

### 2.2 引号类型选择

| 类型 | 使用场景 | 示例 |
|-----|---------|------|
| 双引号「」 | 中文对话（推荐繁体中文） | 「你好」 |
| 双引号"" | 中文对话（简体中文） | "你好" |
| 单引号『』 | 引用中的引用 | 他说：「她说『再见』」 |
| 双引号"" | 内心独白/演讲 | "我要成为最强！" |

**建议**：本文档采用双引号 "" 格式，便于输入和阅读。

### 2.3 引号使用示例

**对话格式**：
```markdown
"阁下是谁？"林凡皱眉问道。
"在下青云门弟子，特来拜访。"对方抱拳道。
```

**内心独白格式**：
```markdown
"没想到竟然是他。"张三心中暗惊，"此人不是在十年前就去世了吗？"
```

**演讲/口号格式**：
```markdown
"天地不仁，以万物为刍狗！"魔君的声音响彻天地。
```

**混合使用**：
```markdown
"父亲，"他跪在地上，"孩儿不孝，无法完成您的遗愿。"
```

### 2.4 禁止省略引号的情况

以下情况**禁止**省略引号：

❌ **错误**：
```markdown
他说我今天必须去。
心理想着这很重要。
演讲的内容是大家好。
```

✅ **正确**：
```markdown
他说："我今天必须去。"
他心理想着："这真的很重要。"
演讲的内容是："大家好！"
```

### 2.5 引号使用检查清单

写作时检查以下问题：
- [ ] 所有对话是否都用引号包裹？
- [ ] 内心独白是否都有引号？
- [ ] 演讲/口号是否都有引号？
- [ ] 引用内容是否都有引号？
- [ ] 引号是否成对出现？

## 3. 规范应用指南

### 3.1 规范优先级

1. **引号使用规范**：高优先级，必须遵守
2. **换段规范**：中高优先级，建议遵守

### 3.2 润色时的检查重点

在 Step 4 润色时，应重点检查：
- [ ] 是否存在"一句话段落"？
- [ ] 换段是否有明确的原因？
- [ ] 对话是否都用了引号？
- [ ] 内心独白是否都用了引号？

### 3.3 常见问题快速修复

**换段问题**：
- 连续3段以内，每段少于30字 → 合并段落
- 同一场景无意义换段 → 删除空行/合并

**引号问题**：
- 对话无引号 → 添加引号
- 引号不成对 → 补全引号
- 内心独白未用引号 → 添加引号

## 5. 角色出场描写规范

### 5.1 首次出场描写

**首次出场角色必须重点描写**：
- 外貌特征：身高、衣着、面容特点
- 气质印象：给主角/读者的第一感觉
- 标志性特征：声音、动作、口头禅等

**非首次出场角色的描写策略**：
- **渐进式揭露**：根据角色在本章的出场方式决定描写深度
- **首次仅闻其声**：本次出场重点描写外貌
- **首次远距离印象**：本次出场可描写近距离细节
- **已有完整印象**：本次出场侧重表情、动作、心理活动

### 5.2 角色动态称呼

**称呼原则**：基于当前叙事视角动态调整对其他角色的称呼

**称呼发展阶段**：
| 阶段 | 条件 | 称呼示例 |
|------|------|---------|
| 陌生人 | 未知姓名 | 公子、姑娘、这位阁下 |
| 初识 | 仅知姓氏 | 林公子、王姑娘、李少侠 |
| 熟人 | 知道全名 | 林少侠、李小姐、张兄 |
| 亲近 | 关系友好 | 兄弟、姐妹、张兄、李兄 |
| 亲密 | 关系密切 | 昵称、小林、小李、心肝 |
| 敌对 | 关系恶劣 | 那人、那个姓林的、刺客 |
| 尊敬 | 辈分/实力高 | 前辈、掌门、长老、宗师 |

**称呼切换规则**：
- 视角切换时保持一致
- 关系变化时称呼随之变化
- 对同一角色的称呼在同一章内应保持稳定

## 6. 道具出场规范

### 6.1 道具首次出现

**首次出现道具**：必须重点描写外观
- 外观特征：形态、颜色、光泽、尺寸、材质感
- 独特标记：纹理、符文、特殊印记
- 气质感觉：给主角/读者的第一感受

### 6.2 道具首次使用

**首次使用道具**：必须重点描写功效
- 功效描述：具体效果和作用
- 使用感受：使用时的感觉（视觉、听觉、触觉）
- 限制与代价：使用限制、副作用、反噬

### 6.3 非首次出现/使用

**已使用过道具**：根据使用场景决定描写重点
- 已熟悉道具：可简写，重点放在当前使用场景的效果
- 进化/升级道具：重点描写变化之处
- 特殊场景：突出道具在当前情境下的特殊表现

## 7. 角色内心独白规范

### 7.1 内心独白格式

**必须使用引号包裹**：
```
"内心独白内容"角色动作/表情
```

### 7.2 内心独白风格匹配

若角色设置了 inner_monologue：

- **语言风格**：简洁有力/幽默/毒舌/直白/温柔等
- **内容特点**：擅长自嘲/喜欢分析/情绪化/理性冷静等
- **独白长度**：长篇大论/简短精悍/点到即止
- **句式特点**：疑问句居多/陈述句为主/感叹强烈

### 7.3 触发场景

在以下场景应使用内心独白：
- 角色内心独白 trigger_scenes 中定义的场景

### 7.4 禁忌

禁止出现：
- 角色内心独白 taboos 中定义的禁忌内容

## 8. 地图移动约束规范

### 8.1 距离约束

| 移动方式 | 速度 | 一时辰行程 |
|---------|------|-----------|
| 正常步行 | 5里/时辰 | 10里 |
| 快速步行 | 8里/时辰 | 16里 |
| 普通马 | 50里/时辰 | 50里 |
| 快马 | 80里/时辰 | 80里 |
| 御剑（筑基） | 200里/时辰 | 1600里 |
| 御剑（金丹） | 500里/时辰 | 4000里 |

### 8.2 方向约束

- 移动方向需与地图方向一致
- 禁止方向错误（如城东写成城西）
- 位置关系需与地图对应

### 8.3 时间约束

- 到达时间需与距离和移动方式匹配
- 禁止"片刻"跨越长距离

### 8.4 路径约束

- 两点间移动需有合理路径
- 禁止直接穿越障碍（山脉、湖泊、禁区）
- 需遵循道路网络

### 8.5 禁止事项

- ❌ 禁止"从东门到西门只需片刻"（距离不合理）
- ❌ 禁止"瞬间移动"除非有传送阵/瞬移技能
- ❌ 禁止方向错误
- ❌ 禁止跳过必经路径

## 9. 外部大纲与参考文件规范

### 9.1 外部大纲文件

**格式**：
```markdown
# 第100章：章节标题

## 目标
主角需要完成什么

## 情节点
- 点1
- 点2

## 角色
- 角色A

## 道具
- 道具A

## 场景
场景描述

## 钩子
- 钩子1
```

**处理规则**：
- 若外部大纲存在，优先使用外部大纲
- 外部大纲可作为原大纲的补充或替代

### 9.2 参考文件

**支持格式**：Markdown、Word、图片、视频

**风格偏离检测**：
- 当参考文件风格与项目整体风格差异超过 30% 时输出警告
- 参考文件不得与项目总体写作风格脱离太多

### 9.3 安全约束

- ✅ 只读取用户明确指定的文件
- ❌ 禁止使用 find/grep 扫描其他文件
- ❌ 禁止使用通配符批量读取
